#+TITLE: Chez-Raylib
#+AUTHOR: Yunoinsky Chen

** Introduction

Raylib bindings for Chez Scheme, built by an s-expression driven binding
generator.

Inspired by Petelliott's [[https://github.com/petelliott/raylib-guile][raylib-guile]].

Thanks to Raysan's outstanding work on [[https://github.com/raysan5/raylib][Raylib]].

λ-0.3 was developed with LLM assistance (DeepSeek V4 + Pi-agent); λ-0.2 was not.

The project uses raylib's own =rlparser= tool to produce sexpr source files,
then feeds them to a Chez Scheme generator that emits =raylib.sls= and
=rlgl.sls= with lazy ~foreign-procedure~ resolution.

** What's included

| Library             | Status         | Description                              |
|---------------------+----------------+------------------------------------------|
| ~(raylib raylib)~    | auto-generated | High-level raylib.h API                  |
| ~(raylib rlgl)~     | auto-generated | rlgl immediate-mode layer                |
| ~(raylib raymath)~  | hand-written   | Matrix interop wrapper around ~(matrix)~ |
| ~(chez-raylib matrix)~ | hand-written | Chez-idiomatic flat-vector matrix lib    |

Auto-generated files are safe to delete and regenerate; the hand-written
libraries keep matrix math Scheme-idiomatic.

** Quick start

1. Install libraylib (e.g. ~brew install raylib~ or build from source).
2. Drop =libraylib.dylib= (or =so= / =.dll=) into ~/Library/chezscheme/raylib/.
3. Build the Scheme libraries:

   #+begin_example
   cd chez-raylib
   scheme --script src/raylib_binding_generator.ss
   echo '(import (chezscheme)) (compile-library "src/raylib.sls")'   | scheme
   echo '(import (chezscheme)) (compile-library "src/rlgl.sls")'     | scheme
   echo '(import (chezscheme)) (compile-library "src/raymath.sls")'  | scheme
   echo '(import (chezscheme)) (compile-library "src/matrix.sls")'   | scheme
   #+end_example

4. Install:

   #+begin_example
   cp src/{raylib,rlgl,raymath,matrix}.{sls,so} ~/Library/chezscheme/raylib/
   #+end_example

5. Run a test (should show a window):

   #+begin_example
   CHEZSCHEMELIBDIRS=~/Library/chezscheme:/usr/local/lib/chezscheme \
     scheme --script test/core/core_basic_window.ss
   #+end_example

** Regenerating bindings after a raylib upgrade

#+begin_example
cd $RAYLIB/tools/rlparser
FORMAT=SEXPR EXTENSION=sexpr make raylib_api.sexpr rlgl_api.sexpr raymath_api.sexpr
cp {raylib,rlgl,raymath}_api.sexpr "$CHEZRAYLIB/src/"
cd "$CHEZRAYLIB"
scheme --script src/raylib_binding_generator.ss
#+end_example

Then recompile and reinstall the four libraries as above.

** Generator source format

The generator (=raylib_binding_generator.ss=) reads sexpr files from
=rlparser=:

- =raylib_api.sexpr= — structs, enums, defines, functions from raylib.h
- =rlgl_api.sexpr= — rlgl immediate-mode layer
- =raymath_api.sexpr= — raymath.h (for verification)

Each sexpr file is a single s-expression with sections: ~(defines ...)~,
~(structs ...)~, ~(enums ...)~, ~(functions ...)~.

The generator handles:
- ~define-ftype~ for C structs (PascalCase-with-hyphens, e.g. ~Camera-3D~)
- ~Make*~ / ~Copy*~ / ~set!~ / ~get~ / ~ref&~ macros per struct
- Lazy ~foreign-procedure~ resolution (load-shared-object at library bottom)
- Type aliases (~Texture2D~ → ~Texture~)
- Opaque pointers (void*)
- Array fields (~char[32]~ → ~(array 32 char)~)
- Preprocessor dedup (rlparser ~#if~ / ~#endif~)
- Integer defines (~RL_TRIANGLES~, ~RL_LINES~, etc.)
- Color defines (~RAYWHITE~, ~RED~, etc.)
- Kebab-case naming for functions (~init-window~, ~load-texture~)
- ~(& Type)~ return type for struct-returning functions
- Variadic function skipping (Chez 10.5 does not support ~...~)

** Chez Scheme idioms used

- ~define-ftype~ for C struct layout; accessor macros generated by the
  generator.
- ~case-lambda~ for multi-arity helpers (e.g. ~make-vector-2~ accepts
  both ~(struct x y z)~ and ~(x y z)~).
- Lazy ~foreign-procedure~ so ~load-shared-object~ lives at the bottom
  of the library and runs once regardless of import order.
- ~(matrix)~ sub-library uses flat vector storage for 2D matrices
  (~index = row * cols + col~). Minimal — only the exports actually
  consumed by ~raymath.sls~ and tests.

** Struct return convention (~(& Type)~)

Functions returning structs (e.g. ~GetMousePosition~ returning ~Vector2~)
use ~(& Vector-2)~ as the foreign-procedure return type. Chez passes a
hidden pointer as the first argument; the binding allocates the buffer,
calls the function, and returns the ftype-pointer. No C shim needed.

** Naming conventions

| C name        | Function                           | ftype             |
|---------------+------------------------------------+-------------------|
| ~InitWindow~  | ~init-window~                      | —                 |
| ~Vector2~     | ~.vector-2~                        | ~Vector-2~        |
| ~Camera2D~    | ~camera-2d~                        | ~Camera-2D~       |
| ~FPS~         | ~fps~                              | —                 |
| ~GetFPS~      | ~get-fps~                          | —                 |

- Functions: kebab-case (hyphen before digit, ~camera-2d~ not ~camera2d~).
- ftype names: PascalCase with hyphens (~Camera-2D~).
- Consecutive uppercase acronyms lowercased (~FPS~ → ~fps~, ~URL~ → ~url~).
- Type aliases resolved: ~Texture2D~ → ~Texture~, ~ModelAnimPose~ → ~Transform~.

** Requirements

- Chez Scheme ≥ 9.5.8 (tested with Chez 10.5.0-pre-release).
- raylib ≥ 6.1 (dynamic library, statically includes rlgl).
