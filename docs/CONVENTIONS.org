#+TITLE: Naming & Idioms

** Naming

| C name        | Function                           | ftype             |
|---------------+------------------------------------+-------------------|
| ~InitWindow~  | ~init-window~                      | —                 |
| ~Vector2~     | ~.vector-2~                        | ~Vector-2~        |
| ~Camera2D~    | ~camera-2d~                        | ~Camera-2D~       |
| ~FPS~         | ~fps~                              | —                 |
| ~GetFPS~      | ~get-fps~                          | —                 |

- Functions: kebab-case (hyphen before digit, ~camera-2d~ not ~camera2d~).
- ftype names: PascalCase with hyphens (~Camera-2D~).
- Consecutive uppercase acronyms lowercased (~FPS~ → ~fps~, ~URL~ → ~url~).
- Type aliases resolved: ~Texture2D~ → ~Texture~, ~ModelAnimPose~ → ~Transform~.

** Struct return (~(& Type)~)

Functions returning structs (e.g. ~GetMousePosition~ returning ~Vector2~)
use ~(& Vector-2)~ as the foreign-procedure return type. Chez passes a
hidden pointer as the first argument; the binding allocates the buffer,
calls the function, and returns the ftype-pointer. No C shim needed.

** Chez Scheme idioms

- ~define-ftype~ for C struct layout; accessor macros generated by the
  generator.
- ~case-lambda~ for multi-arity helpers (e.g. ~make-vector-2~ accepts
  both ~(struct x y z)~ and ~(x y z)~).
- Lazy ~foreign-procedure~ so ~load-shared-object~ lives at the bottom
  of the library and runs once regardless of import order.
- ~(matrix)~ sub-library uses flat vector storage for 2D matrices
  (~index = row * cols + col~). Minimal — only the exports actually
  consumed by ~raymath.sls~ and tests.
